STEP 4: SET UP ORDER ARCHIVAL (OPTIONAL)
═══════════════════════════════════════════════════════════════════════════

If you want automatic order archival after 1 week:

1. In your Google Sheet, duplicate the VSM_Execution sheet
2. Rename the duplicate to: VSM_Execution_Archive
3. Add 3 columns at the end:
   - Archived Date
   - Archived By
   - Days Since Dispatch
4. Clear all data rows (keep headers only)

5. In Apps Script editor, find function: setupDailyArchivalTrigger
6. Click the ▶️ Run button next to it
7. Authorize if prompted
8. Check "Triggers" tab (clock icon ⏰) to verify trigger was created

WHAT EACH FEATURE DOES
═══════════════════════════════════════════════════════════════════════════

🔒 PROCESS SEQUENCE ENFORCEMENT:
   - Users can only enter data for unlocked processes
   - Processes unlock when dependencies are met:
     * Strict: Previous process 100% complete (has actualEndDate)
     * Partial: Previous process started (has actualStartDate)
   - Visual indicators: ✓ Done | ⏳ In Progress | 🔓 Available | 🔒 Locked

✅ TRANSACTIONAL COMPLETION FIX:
   - Cutting, Sewing, Washing, Finishing, Inspection now complete properly
   - When cumulative quantity reaches order quantity:
     * actualEndDate is set automatically
     * Process Status changes to "Completed"
   - No more stuck "In Progress" status

📦 ORDER ARCHIVAL SYSTEM:
   - Automatically moves old orders to archive
   - Runs daily at 1 AM
   - Archives orders 1 week after dispatch
   - Keeps active tracking clean and fast

🔍 INSPECTION 10% TRACKING:
   - Inspection added to transactional processes
   - Tracks cumulative received quantity from Finishing
   - Inspects 10% sample (auto-calculated suggestion)
   - Shows progress: Received / Inspected / Remaining

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

❌ "Process Status Endpoint not working"
   → Check Apps Script logs: View → Logs
   → Verify deployment URL is correct in your frontend
   → Test URL: YOUR_URL?action=getProcessStatus&ocNo=TEST_OC_NUMBER

❌ "Transactional processes still stuck at In Progress"
   → Check if cumulative quantity = order quantity
   → Look at Apps Script logs for "getActualDates" messages
   → Verify WIP_QTY column (AV) has formula: =MAX(0,AT-AU)

❌ "Archival not working"
   → Check if VSM_Execution_Archive sheet exists
   → Verify trigger is set up (Triggers tab, clock icon)
   → Run archiveOldOrders() manually to test
   → Check logs for error messages

❌ "Inspection not showing 10% fields"
   → Restart your dev server: npm run dev
   → Clear browser cache
   → Check TaskUpdatePageEnhanced.tsx has Inspection in transactional list
