PROBLEM: Cutting, Sewing, etc. show "In Progress" even at 1000/1000

   TO:   rowData[COL.ACTUAL_END - 1] = actualDates.actualEnd || '';

═══════════════════════════════════════════════════════════════
FEATURE #2: PROCESS SEQUENCE (15 MIN)
═══════════════════════════════════════════════════════════════

PROBLEM: Users can skip processes, no department isolation

FIX: ADD FUNCTION + UPDATE doGet()

1. Open google-apps-script/PROCESS_STATUS_ENDPOINT.gs
2. Copy the getProcessStatus() function
3. Paste at END of your Apps Script file
4. Find your doGet() function
5. Add this inside doGet():

   if (action === 'getProcessStatus') {
     const ocNo = e.parameter.ocNo;
     if (!ocNo) {
       return ContentService.createTextOutput(JSON.stringify({
         success: false,
         error: 'OC Number is required'
       })).setMimeType(ContentService.MimeType.JSON);
     }
     
     const result = getProcessStatus(ocNo);
     return ContentService.createTextOutput(JSON.stringify(result))
       .setMimeType(ContentService.MimeType.JSON);
   }

6. Save (Ctrl+S)
7. Deploy → Manage deployments → Edit → New version → Deploy

RESULT: Users can only select unlocked processes! 🔒

═══════════════════════════════════════════════════════════════
FEATURE #3: ORDER ARCHIVAL (15 MIN)
═══════════════════════════════════════════════════════════════

PROBLEM: Dashboard cluttered with old dispatched orders

FIX: CREATE ARCHIVE SHEET + ADD FUNCTIONS + SET TRIGGER

STEP 1: Create Archive Sheet (5 min)
1. Open Google Sheet
2. Right-click "VSM_Execution" tab
3. Click "Duplicate"
4. Rename to "VSM_Execution_Archive"
5. Add 3 columns at end:
   - AX: Archived Date
   - AY: Archived By
   - AZ: Days Since Dispatch
6. Delete all data rows (keep headers)

STEP 2: Add Functions (5 min)
1. Open google-apps-script/ORDER_ARCHIVAL.gs
2. Copy ALL functions
3. Paste at END of your Apps Script file
4. Save (Ctrl+S)
5. Deploy → Manage deployments → Edit → New version → Deploy

STEP 3: Set Up Trigger (5 min)
1. In Apps Script, select function: setupDailyArchivalTrigger
2. Click Run (▶️)
3. Authorize if prompted
4. Check "Triggers" tab (⏰ icon)
5. Verify trigger exists: archiveOldOrders - Day timer - 1-2am

RESULT: Orders automatically archived 1 week after dispatch! 📦
