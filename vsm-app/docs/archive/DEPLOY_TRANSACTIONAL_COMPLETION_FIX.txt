PROBLEM:
Cutting, Sewing, Washing, Finishing, Inspection show "In Progress"
even when fully completed (cumulative qty = order qty)

SOLUTION:
Use actual end date from getActualDates() when WIP = 0

───────────────────────────────────────────────────────────────
CRITICAL CHANGE (ONE LINE!)
───────────────────────────────────────────────────────────────

In your Code_WithCalculations_FIXED_V2.gs file, around line 1221:

BEFORE (WRONG):
  rowData[COL.ACTUAL_END - 1] = ''; // ❌ Always empty!

AFTER (CORRECT):
  rowData[COL.ACTUAL_END - 1] = actualDates.actualEnd || ''; // ✅

───────────────────────────────────────────────────────────────
HOW IT WORKS
───────────────────────────────────────────────────────────────

For Transactional Processes (Cutting, Sewing, etc.):

1. User enters daily quantities
2. System calculates:
   - CUM_ACHIEVED = SUMIFS of all quantities
   - WIP_QTY = ORDER_QTY - CUM_ACHIEVED

3. When WIP_QTY <= 0:
   - getActualDates() finds the last entry date
   - This date is set as Actual End Date
   - Status is calculated: "Completed - On Time" or "Completed - Delayed"

───────────────────────────────────────────────────────────────
TROUBLESHOOTING
───────────────────────────────────────────────────────────────

Issue: Still showing "In Progress" after WIP = 0
Fix: 
  - Check Apps Script logs for errors
  - Verify COL.WIP_QTY = 48 (Column AV)
  - Verify COL.ENTRY_DATE = 44 (Column AR)
  - Check WIP formula in sheet: =MAX(0, AT-AU)

Issue: Actual End Date not set
Fix:
  - Verify getActualDates() function exists
  - Check that actualDates.actualEnd is being used (not '')
  - Look at Apps Script logs for "Last End (WIP=0)"

Issue: Wrong completion date
Fix:
  - getActualDates() finds LAST date where WIP = 0
  - If multiple entries after completion, uses latest
  - Check ENTRY_DATE column has correct dates
